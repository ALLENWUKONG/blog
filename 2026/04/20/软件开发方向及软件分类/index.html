IT类 专业、 行业

软件：程序+数据

软件的分类：

1.系统软件>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>{独领风骚}【C/C++/C#】

1）驱动程序：硬件和软件沟通的桥梁

2）操作系统：win11 linux macos

2.1）总结：大企业，超大公司 eg：Google、微软、IBM、Apple

要求：学历985/211 要求对系统底层非常熟悉

2.C/S架构软件>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>{大多是是游戏}

client 客户端 需要下载、安装（QQ、微信、lol）

server 服务端

2.1）公司有个超级大电脑：服务器——存放QQ账号，ID账号，密码……（我们几乎不能看见的东西，客户与QQ交互的地方【要有网络】）

3.B / S 架构软件>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>{独领风骚}【软件就业很重要；岗位多】

browser（浏览器）

server （服务端）

3.1）都是网站，比如：淘宝、京东、当当……（PC端，手机不在讨论范围）；也能登录，没必要下载客户端

总结：以网站的形式与客户交互的软件（PC端）

3.2) 前端/后端

前端：网页编写（三要素：HTML,CSS,JavaScript）

后端：Java/go

4.移动端（大部分都是C/S）

Android Java kotlin >>>>>Flutter Dart web端，所有端都能用

ios OC

app application

找工作基本是凉的，比如微信小程序【虽然更偏向B/S】（在夹缝中生存，基本没有前景）

5.系统商店应用

win8以后有个window商店（生态很垃圾，我一般还是去网站下载，而且功能不完整）

属于UWP（垃圾实锤）

Ubuntu（里面的商店类似）

MacOS Store final cut>>>

Unix homebrew

手机商店太成熟了：apple app store / android googleplay

总结：不如GitHub

初学者比较关注计算机语言

语言、库、框架、流行技术>>>>>>>>>>>>>>>>>>>>>>>>>总称：技术栈（不是绝对的，是相对的）
